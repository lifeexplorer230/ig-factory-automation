### Запись 2.0.2 — Критические открытия MoreLogin API

**Дата:** 24.02.2026
**Фаза:** 2.1
**Статус:** ⚠️ Добыто через отладку

**Открытие 1: client_id — INTEGER, не строка**
- ❌ `"client_id": "1689105551924663"` → code 1001, "invalid client_id"
- ✅ `"client_id": 1689105551924663` (без кавычек)
- Касается: `/oauth2/token`

**Открытие 2: Пути БЕЗ /api/ префикса**
- ❌ `/api/cloudphone/page` → 404
- ✅ `/cloudphone/page`
- Все пути: `/cloudphone/...`, `/proxyInfo/...`, `/oauth2/...`

**Открытие 3: proxyInfo/delete — голый JSON array**
- ❌ `{"ids": [123, 456]}` → ошибка
- ✅ `[123, 456]` (тело запроса = просто массив integers)

**Открытие 4: app/install — appVersionId, не packageName**
- ❌ `{"packageName": "com.instagram.android"}` → не работает
- ✅ `{"appVersionId": "1682134957917431"}`
- appVersionId ищется через `/cloudphone/app/page` с `appName` в теле
- Без `appName` в теле — возвращает только 10 дефолтных приложений!

**Открытие 5: Прокси — proxyProvider:0 обязателен**
- Без `proxyProvider:0` credentials (user/pass) не сохраняются
- SOCKS5: `proxyType:2`

**Открытие 6: envStatus значения**
- 0=New, 1=Failed, 2=Stop, 3=Starting, 4=Running, 5=Resetting
- Ждать нужно именно `status=4 AND adbInfo.adbIp непустой`

**Дата:** 24.02.2026
**Фаза:** 2.3 — 2.4
**Статус:** ⚠️ Добыто через отладку

**Открытие 7: ADB keeper-сессия обязательна**
- Обычный `adb shell <cmd>` → "error: closed"
- Нужно открыть keeper: `adb shell <password>` (интерактивный процесс)
- Пока keeper жив — параллельные `adb shell` команды работают нормально
- После создания телефона сразу вызывать `/cloudphone/updateAdb` с `enableAdb:true`

**Открытие 9: Логин — три сценария верификации**
- A) Стандартный TOTP: "Enter the 6-digit code" → вводим код из pyotp
- B) Device approval: "Check your notifications" → "Try another way" → "Authentication app" → TOTP
- C) Уже залогинен → пропускаем (проверка по `tab_bar` / `bottom_tray`)
