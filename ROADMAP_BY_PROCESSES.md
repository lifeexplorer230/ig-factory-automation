# 🎬 Дорожная карта по процессам (а не по дням)

> Вместо "День 1, День 2..." — делаем по процессам и критериям.  
> Когда задача готова → переходим к следующей. Параллельно прогреваем аккаунты.

---

## 🏗️ АРХИТЕКТУРА ПРОЕКТА (Параллельные потоки)

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  ПОТОК 1: КОНТЕНТ-ФАБРИКА          ПОТОК 2: INSTAGRAM-ФАБРИКА │
│  (Генерация видео)                  (Прогрев аккаунтов)         │
│                                                                 │
│  ├─ Nano Banana API                 ├─ Создание аккаунтов      │
│  ├─ Kling AI API                    ├─ Добавление прокси       │
│  ├─ Google Drive                    ├─ Создание телефонов      │
│  ├─ Content Pipeline                ├─ Логин в Instagram       │
│  └─ Очередь видео (Redis)           └─ Прогрев (Reels)         │
│                                                                 │
│  Результат: 50+ видео в очереди     Результат: готовые        │
│                                      аккаунты к публикации      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                    ПОТОК 3: ПУБЛИКАЦИЯ
                    (Когда оба потока готовы)
                    
                    ├─ Берём видео из очереди
                    ├─ Публикуем на аккаунты
                    ├─ Собираем метрики
                    └─ Масштабируем
```

## 📊 ПОТОК 2: INSTAGRAM-ФАБРИКА (Прогрев)

### Этап 2.1: Подготовка инфраструктуры (Критерий: 5 телефонов готово)

**Что нужно:**
- [ ] MoreLogin API ключ ✅
- [ ] Прокси (SOCKS5)
- [ ] Бюджет на закупку аккаунтов

**Что делаем:**
1. Создаём скрипт `morelogin_client.py`
2. Создаём 5 виртуальных телефонов (CP-1..CP-5)
3. Добавляем прокси
4. Устанавливаем Instagram на каждый

**Критерий завершения:**
- ✅ 5 телефонов созданы
- ✅ 5 прокси добавлены
- ✅ Instagram установлен на всех
- ✅ ADB доступен для каждого

---

### Этап 2.2: Закупка и подготовка аккаунтов (Критерий: 5 аккаунтов готово)

**Что делаем:**
1. Закупляем 5 Instagram аккаунтов
2. Сохраняем credentials
3. Подготавливаем для логина

**Критерий завершения:**
- ✅ 5 аккаунтов куплены
- ✅ Credentials сохранены
- ✅ Email/пароль/TOTP есть
- ✅ Готовы к логину

---

### Этап 2.3: Логин на аккаунты (Критерий: все аккаунты залогинены)

**Что делаем:**
1. Создаём скрипт `multi_account_login.py`
2. Логинимся через ADB + UIAutomator
3. Обрабатываем TOTP и device-approval
4. Сохраняем session cookies

**Критерий завершения:**
- ✅ 5 аккаунтов залогинены
- ✅ Session cookies сохранены
- ✅ Нет ошибок логина
- ✅ Аккаунты готовы к использованию

**Если не работает:**
- Проверяем TOTP
- Проверяем device-approval (может потребоваться подтверждение)
- Смотрим логи ADB
- Пишем Климу (может быть блокировка Instagram)

---

### Этап 2.4: Прогрев аккаунтов (Критерий: аккаунты "живут" в Instagram)

**Что делаем:**
1. Создаём скрипт `ig_warmup.py` (уже есть из Phase 0)
2. Запускаем прогрев на 5 аккаунтах
3. Смотрим, нет ли банов

**Прогрев включает:**
- Просмотр 25 Reels (4:30-5:00 мин)
- 40% смотрим полностью, 60% пролистываем
- 20% шанс лайка
- Jitter на тапы (уникальность)

**Критерий завершения:**
- ✅ 5 аккаунтов прогреты
- ✅ Нет банов
- ✅ Аккаунты "живые" (видны в Instagram)
- ✅ Готовы к публикации

**Если аккаунты получают бан:**
- Анализируем причину (IP, поведение, время)
- Меняем стратегию прогрева
- Закупляем новые аккаунты
- Пишем Климу

**Критерий завершения:**

## 💡 Что я не учёл (и что нужно добавить)

### 1. Резервные копии и восстановление
- Что если сервер упадёт?
- Как восстановить session cookies?
- Как восстановить очередь видео?

**Решение:** Добавить скрипт `backup_manager.py`
- Ежедневная резервная копия PostgreSQL
- Резервная копия session cookies
- Резервная копия очереди Redis

### 2. Мониторинг и алерты
- Что если публикация упала?
- Что если аккаунты получают бан?
- Что если контент-фабрика зависла?

**Решение:** Добавить скрипт `monitoring_alerts.py`
- Проверка статуса каждый час
- Алерты в Telegram при ошибках
- Автоматический перезапуск при сбое

### 3. Версионирование контента
- Какие видео уже публиковались?
- Не публиковать одно видео дважды?
- История всех публикаций?

**Решение:** Добавить таблицу `published_videos` в PostgreSQL
- Отслеживать все опубликованные видео
- Не публиковать дубликаты
- История для аналитики

### 4. A/B тестирование
- Какие подписи работают лучше?
- Какие видео работают лучше?
- Какое время публикации лучше?

**Решение:** Добавить скрипт `ab_testing.py`
- Тестировать разные подписи
- Тестировать разные видео
- Тестировать разные времена

### 5. Управление бюджетом
- Сколько потратили на API?
- Сколько потратили на прокси?
- ROI на публикацию?

**Решение:** Добавить скрипт `budget_tracker.py`
- Отслеживать расходы
- Считать ROI
- Оптимизировать бюджет

### 6. Обработка ошибок и fallback
- Что если Nano Banana API недоступен?
- Что если Kling AI медленный?
- Что если Google Drive упал?

**Решение:** Добавить fallback логику
- Retry с exponential backoff
- Использовать cached результаты
- Очередь для повторной обработки

### 7. Версионирование и откат
- Что если новая версия сломала систему?
- Как откатиться на старую версию?
- История всех изменений?

**Решение:** Добавить git для всех скриптов
- Каждое изменение в git
- Теги для стабильных версий
- Возможность отката

### 8. Логирование и аудит
- Кто что делал и когда?
- Все ошибки логируются?
- Можем ли мы воспроизвести проблему?

**Решение:** Добавить структурированное логирование
- JSON логи для парсинга
- Уровни логирования (DEBUG, INFO, WARNING, ERROR)
- Централизованное хранилище логов

### 9. Масштабирование на несколько серверов
- Что если один сервер не справляется?
- Как распределить нагрузку?
- Как синхронизировать состояние?

**Решение:** Добавить поддержку распределённой системы
- Redis как центральное хранилище состояния
- Несколько рабочих процессов
- Load balancer для распределения

### 10. Юридические и этические вопросы
- Нарушаем ли мы TOS Instagram?
- Нужны ли согласования?
- Какие риски?

**Решение:** Документировать все риски
- Проверить TOS Instagram
- Проверить локальное законодательство
- Обсудить с Климом
