## 🎯 Процесс работы (TDD)

### Для каждой фазы:

```
1️⃣ НАПИШИ ТЕСТ
   └─ Тест = критерий завершения фазы
   └─ Файл: /home/roma/tests/test_X_X_name.py

2️⃣ ЗАПУСТИ ТЕСТ (он падает)
   └─ pytest /home/roma/tests/test_X_X_name.py -v
   └─ Результат: FAILED ❌

3️⃣ НАПИШИ СКРИПТ
   └─ Скрипт должен пройти тест
   └─ Файл: /home/roma/scripts/script_name.py

4️⃣ ЗАПУСТИ ТЕСТ (он проходит)
   └─ pytest /home/roma/tests/test_X_X_name.py -v
   └─ Результат: PASSED ✅

5️⃣ ДОКУМЕНТИРУЙ РЕЗУЛЬТАТ
   └─ Открой /home/roma/PROJECT_JOURNAL.md
   └─ Добавь запись о фазе
   └─ Что делали, результат, гипотеза, реальность, урок

6️⃣ ПЕРЕХОДИ К СЛЕДУЮЩЕЙ ФАЗЕ
   └─ Повтори шаги 1-5
```

## 🚀 Быстрый старт

### День 1
```bash

cat /home/roma/TESTING_STRATEGY.md
```
